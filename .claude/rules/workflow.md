# Workflow rules

- Always use plan mode before starting any feature that touches more than one file
- After implementing a feature, tell me exactly how to test it manually
- If you hit a Supabase or Stripe error, show me the full error message before attempting a fix
- Commit frequently with short imperative messages (e.g. "add invoice draft endpoint")
- Never install a new npm package without telling me what it does and why we need it
